## Cache-Control Headers by Prefix

| Prefix | Cache-Control | Reason |
|--------|---------------|--------|
| `frontend/_next/static/*` | `public, max-age=31536000, immutable` | Content-hashed filenames — safe to cache forever |
| `frontend/**` (HTML, images) | `public, max-age=0, must-revalidate` | Always revalidate; users get fresh HTML immediately |

These headers are set by `aws s3 sync --cache-control "..."` in the CI/CD workflow and become the `Cache-Control` response header when CloudFront fetches from S3.

## Bucket Policy

The bucket policy has four statements:

### 1. `AllowCloudFrontOAC` — CloudFront reads frontend assets

```json
{
  "Sid": "AllowCloudFrontOAC",
  "Effect": "Allow",
  "Principal": {"Service": "cloudfront.amazonaws.com"},
  "Action": "s3:GetObject",
  "Resource": "arn:aws:s3:::group-2-286664220957-us-east-1-an/frontend/*",
  "Condition": {
    "StringEquals": {
      "AWS:SourceArn": "arn:aws:cloudfront::286664220957:distribution/E3MRCL1361H3LW"
    }
  }
}
```

Only the specific CloudFront distribution `E3MRCL1361H3LW` can read `frontend/*`. Any other caller (including another CloudFront distribution or direct S3 URL) gets denied. This is enforced by the `aws:SourceArn` condition.

### 2. `AllowEC2RoleSync` — EC2 instances pull deployment artifacts

```json
{
  "Sid": "AllowEC2RoleSync",
  "Effect": "Allow",
  "Principal": {"AWS": "arn:aws:iam::286664220957:role/group-2-ec2-ssm-role"},
  "Action": ["s3:GetObject", "s3:ListBucket"],
  "Resource": [
    "arn:aws:s3:::group-2-286664220957-us-east-1-an",
    "arn:aws:s3:::group-2-286664220957-us-east-1-an/frontend/*"
  ]
}
```

Allows the EC2 IAM role to run `aws s3 sync s3://bucket/frontend/ /var/www/html/` during deployments triggered via SSM.

### 3. `AllowALBLogs` — ALB writes access logs

```json
{
  "Sid": "AllowALBLogs",
  "Effect": "Allow",
  "Principal": {"AWS": "arn:aws:iam::127311923021:root"},
  "Action": "s3:PutObject",
  "Resource": "arn:aws:s3:::group-2-286664220957-us-east-1-an/logs/alb/*"
}
```

AWS account `127311923021` is the ELB service account for `us-east-1`. This allows the ALB to write access logs to `logs/alb/`.

### 4. `DenyUnencryptedUploads` — Enforce encryption at upload

```json
{
  "Sid": "DenyUnencryptedUploads",
  "Effect": "Deny",
  "Principal": "*",
  "Action": "s3:PutObject",
  "Resource": "arn:aws:s3:::group-2-286664220957-us-east-1-an/*",
  "Condition": {
    "Null": {"s3:x-amz-server-side-encryption": "false"},
    "StringNotEquals": {"s3:x-amz-server-side-encryption": "AES256"}
  }
}
```

Denies any `PutObject` that does not specify `x-amz-server-side-encryption: AES256`. The CI/CD `aws s3 sync` commands pass `--sse AES256` to satisfy this condition.
